# Security Policy

## Reporting a vulnerability

Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.

Instead, report them privately through GitHub's
[private security advisory](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)
flow for this repository ("Security" tab -> "Report a vulnerability").

Please include:

- a description of the issue and its impact,
- steps to reproduce or a proof of concept,
- affected version or commit, and
- any suggested remediation.

We aim to acknowledge a report within a few business days and will keep you
updated as we investigate and prepare a fix. We appreciate coordinated
disclosure and will credit reporters who wish to be named once a fix ships.

## Supported versions

Primer is pre-1.0 and under active development. Security fixes target the
`main` branch; there is no long-term-support branch yet.
